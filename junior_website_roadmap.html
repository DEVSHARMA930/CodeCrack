Code Editor — Detailed Plan

C / C++ / Python support Syntax highlighting Output console Run button Clear & reset Font size control Sample starter code

The editor will feel like a lightweight VS Code / Dev C++ in the browser. A language selector at the top lets users switch between C, C++, and Python. The left panel is the code area (Monaco Editor or CodeMirror), and the right/bottom panel shows the program output. A "Run ▶" button compiles and executes via a sandboxed backend API (e.g. Judge0 or Piston). Error messages and stdout both appear in the console.

Design Direction

Color schemeA clean white/off-white base with one accent color (indigo or teal). Dark mode variant.
TypographyA readable sans-serif (Inter or Poppins). Large headings, comfortable body size (~15–16px).
LayoutSidebar navigation on desktop, bottom tab bar on mobile. Content area takes up the full width.
Cards & surfacesSubject notes and papers shown as cards in a grid. Subtle borders, no heavy shadows.
Hero sectionHome page has a welcoming banner with a short tagline, search bar, and quick links to each section.
Code editor UIDark-themed editor pane always (easier on eyes), even when the rest of the site is in light mode.

Build Phases

Phase 1 — Foundation
Set up project structure, routing, navbar, home page, dark mode toggle, and basic layout.
Phase 2 — Content
Build Notes and PYQ pages. Upload initial files. Add subject filters and PDF viewer.
Phase 3 — Editor
Integrate Monaco Editor. Connect Judge0/Piston API. Build output console and language switcher.
Phase 4 — Polish
Add global search, mobile responsiveness, loading states, and deploy to Vercel / Netlify.
